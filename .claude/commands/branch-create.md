# 기능 개선 브랜치 생성 + 개발 플랜 수립

기능 개선을 위한 브랜치를 생성하고 실행 가능한 개발 플랜을 수립합니다.

## 입력

$ARGUMENTS: 기능명과 개선 목표/방향
- 예시: "coupon 만료 임박 알림 기능 추가"
- 예시: "payment 결제 실패 시 재시도 로직 개선"

## 작업 순서

### 1. 입력 파싱
- 첫 번째 단어 → 기능명 (예: coupon, payment, auth)
- 나머지 내용 → 개선 목표/방향

### 2. 브랜치 버전 확인 및 생성
```bash
# 기존 브랜치 확인
git branch -a | grep "{기능명}-v"

# 버전 결정: 없으면 v1, 있으면 다음 버전 (v2, v3...)
# 새 브랜치 생성
git checkout -b {기능명}-v{버전}
```

### 3. 현재 코드 분석
- 프로젝트에서 해당 기능 관련 파일 탐색
- 디렉토리 구조, 주요 파일, 의존성 파악
- 현재 구현 상태 요약

### 4. 개선 플랜 문서 생성
`.plans/{기능명}-v{버전}.md` 파일을 아래 형식으로 생성:

```markdown
# {기능명} v{버전} 개선 플랜

## 개선 목표
{사용자가 입력한 개선 목표/방향}

## 현재 상태 분석
- 관련 파일 목록
- 현재 동작 방식 요약
- 개선이 필요한 부분

## 구현 계획

### Phase 1: {단계명}
- [ ] 작업 1
- [ ] 작업 2

### Phase 2: {단계명}
- [ ] 작업 3
- [ ] 작업 4

## 예상 변경 파일
- `path/to/file1.ts` - 변경 이유
- `path/to/file2.ts` - 변경 이유

## 테스트 계획
- [ ] 테스트 항목 1
- [ ] 테스트 항목 2
```

### 5. 결과 출력
- ✅ 생성된 브랜치명
- 📄 플랜 문서 경로
- 🚀 첫 번째로 실행할 작업 제안
